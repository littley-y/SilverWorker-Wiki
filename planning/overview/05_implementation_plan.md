## 1. 핵심 전략: 무엇을 버릴까?

2주 내 동작하는 데모를 위해 다음을 **완전히 제외**합니다:

| 제외 항목 | 이유 |
|---|---|
| FCM 푸시 알림 | 데모 수준에서는 불필요. 추후 추가 |
| 찜하기(스크랩) | 핵심 흐름(검색→지원)에 없음 |
| 지원 취소 | 데모에서는 지원 완료까지만 |

| iOS 빌드 | Android 에뮬레이터/기기로만 데모 |
| CI/CD 파이프라인 | 수동 빌드로 대체 |
| Cloud Functions Cron | API 미승인 시 Mock 데이터 사용 |
| API 캐싱 레이어 | 데모 트래픽에서는 불필요 |
| 접근성 상세 검증 | 기본 텍스트 크기만 확대 |
| 오프라인 캐시 | 온라인 가정 |

## 4. 2주 안에 들어가지 않는 것 (명시적 제외)

다음은 2주 데모 이후 추가 개발 대상입니다:

| 기능 | 제외 이유 | 추정 추가 소요 |
|---|---|---|
| 푸시 알림 (FCM) | 데모에서는 화면 내 상태 변화로 충분 | +2일 |
| 찜하기 | 핵심 흐름 아님 | +1일 |
| 프로필 수정 | 데모는 1회 등록 가정. 조회만 구현 | +0.5일 |
| 다중 지역 선택 | 2주 내 단일 선택으로 충분히 데모 가능 | +0.5일 |
| 오프라인 캐시 | 온라인 환경 가정. 2차에서 flutter_cache_manager 적용 | +1일 |
| Cloud Functions Cron | 데모에서는 수동/1회성 데이터로 충분 | +2일 |
| API 캐싱 | 데모 트래픽 낮음 | +1일 |
| iOS 빌드 | Android 데모로 충분 | +2일 |
| 접근성 상세 (TalkBack) | 기본 텍스트 크기만 적용 | +2일 |
| 설정 화면 (글꼴 크기 변경) | 기본 20pt 고정 | +1일 |
| 마스코트 캐릭터 | 2주 데모에서는 이미지 에셋 미포함. 기획서상 차별화 포인트로 2차 구현 | +2일 |
| 회원 탈퇴 | 데모 불필요 | +0.5일 |
| 비밀번호 찾기 | 휴대폰 인증 재사용으로 대체 가능 | +1일 |

## 5. 데모 시나리오 (시연 흐름)

[시작] 앱 실행 -> 스플래시 화면
  ->
[로그인] 휴대폰 번호 입력 -> SMS 인증코드 입력 (자동 완성)
  ->
[프로필 등록] 이름, 지역(서울 종로구), 간단 경력 입력
  ->
[메인 홈] 공고 목록 (카드 10개 표시)
  ->
[필터 적용] 지역: 종로구 -> 직종: 경비/관리
  ->
[공고 상세] 업무 강도 배지 확인 (서있기/보통)
  ->
[지원하기] 자기소개 입력 (경비 업무 경험 있습니다.) -> 지원 완료
  ->
[마이페이지] 지원 내역 확인 (1건, 상태: 접수)
  ->
[로그아웃] 시연 종료

예상 시연 시간: 3~5분

## 8. 성공 기준 (Definition of Done)

2주 후 다음 흐름이 끊김 없이 동작하면 성공입니다:

1. 앱 실행 후 로그인 화면 표시 (3초 이내)
2. 휴대폰 인증 완료 후 프로필 등록 가능
3. 메인 화면에 공고 목록 표시 (최소 10개)
4. 필터 적용 시 목록 변경
5. 공고 상세에서 업무 강도 배지 확인
6. 지원 버튼 클릭 후 Firestore에 저장
7. 마이페이지에서 지원 내역 확인

배포는 필요 없습니다. APK 파일 1개만 생성되어 실제 기기에서 설치 및 실행 가능하면 충분합니다.
