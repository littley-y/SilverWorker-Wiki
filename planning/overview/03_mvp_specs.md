# 03. SilverWorkerNow MVP 기능 명세서

> 범위: AI 기능 제외, 핵심 구인구직 매칭 기능 중심
> 작성 일자: 2026-04-24

---

## 1. MVP 정의

**MVP (Minimum Viable Product)**: 시니어 사용자가 고용24 공공데이터 기반의 일자리를 검색하고, 간단한 이력서로 지원할 수 있는 최소 기능 집합.

**제외 기능 (2차 이후)**:
- AI 음성 이력서 자동 변환
- AI 직무 적합도 추론
- AI 안면 인식 출퇴근
- AI 리뷰 요약
- AI 컨디션 기반 매칭

---

## 2. 사용자 유형

| 유형 | 설명 | MVP 포함 여부 |
|---|---|---|
| **구직자 (시니어)** | 60세 이상 일자리 희망자 | 포함 |
| **구인자 (기업/소상공인)** | 일자리 공고 등록 및 지원자 관리 | 제외 (2차) |
| **관리자** | 공고 승인, 사용자 관리, 통계 | 제외 (2차) |

## 3. 기능 목록

### 3.1 인증 및 회원가입

| ID | 기능 | 상세 | 우선순위 |
|---|---|---|---|
| AUTH-01 | 휴대폰 번호 인증 가입 | Firebase Phone Auth. 국내 휴대폰 번호 입력 → SMS 인증코드 입력 → 가입 완료 | P0 |
| AUTH-02 | 로그인 / 로그아웃 | 휴대폰 번호 + 인증코드 재발송 방식. 자동 로그인(30일) | P0 |
| AUTH-03 | 간편 프로필 등록 | 이름, 생년월일, 거주 지역(시/구), 간단한 경력(텍스트 500자 내외) | P0 |
| AUTH-04 | 게스트 로그인 (폰백) | 휴잭폰 인증 실패/SMS 지연 시 간편 PIN 번호 또는 게스트 모드로 데모 진행 가능 | P1 |

### 3.2 공고 검색 및 조회

| ID | 기능 | 상세 | 우선순위 |
|---|---|---|---|
| JOB-01 | 공고 목록 조회 | 고용24 API 연동. 최신 공고 50개 페이징 로드. Firebase Functions 프록시 통해 JSON 수신 | P0 |
| JOB-02 | 지역 필터 | 사용자 거주 지역 기준 반경(예: 동일 구, 인접 구) 필터. **2주 데모에서는 단일 선택**로 제한 | P0 |
| JOB-03 | 직종 필터 | 단순 노무, 경비/관리, 청소/미화, 서비스, 사무/문서 등 대분류 필터 | P0 |
| JOB-04 | 근무 형태 필터 | 파트타임, 일용직, 단기, 정규직 | P1 |
| JOB-05 | 공고 상세 조회 | 근무지, 급여, 근무시간, 복리후생, 업무 내용, 업무 강도 배지 표시 | P0 |
| JOB-06 | 공고 스크랩/찜 | 관심 공고를 찜 목록에 저장. 로컬 또는 Firestore 저장 | P1 |

### 3.3 업무 강도 시각화 (세이프티 큐레이션)

| ID | 기능 | 상세 | 우선순위 |
|---|---|---|---|
| SAFE-01 | 체력 소모 배지 | 공고 상세에 '계속 서있기', '무거운 짐 운 반기', '좌식 업무' 등 아이콘 배지 표시 | P0 |
| SAFE-02 | 체력 등급 표시 | 업무 강도를 '가벼움 / 보통 / 무거움' 3단계로 시각화 | P0 |
| SAFE-03 | 사용자 체력 조건 등록 | "무릎이 안 좋음", "오래 서기 어려움" 등 체크리스트. 공고 검색 시 필터링 가중치에 반영 (정적 필터) | P1 |

### 3.4 지원 기능

| ID | 기능 | 상세 | 우선순위 |
|---|---|---|---|
| APP-01 | 공고 지원 | 지원하기 버튼 클릭 → 간단한 지원서(자기소개 200자) 제출 → 지원 완료 알림 | P0 |
| APP-02 | 지원 내역 조회 | 내가 지원한 공고 목록 및 상태(접수/검토/합격/불합격) 조회 | P0 |
| APP-03 | 지원 취소 | 검토 전 상태에서 지원 취소 가능 | P1 |
| APP-04 | 오터치 방어 | 지원 버튼 클릭 시 1.5초 쿨다운 적용. **버튼 회색 처리 + 로딩 스피너 표시**로 시각적 피드백 제공 | P0 |

### 3.5 마이페이지

| ID | 기능 | 상세 | 우선순위 |
|---|---|---|---|

| MY-01b | 내 프로필 수정 | 위 항목 편집. **2주 데모 제외 → 2차 개발** | P1 |

| MY-04 | 알림 설정 | 새 공고 알림, 지원 결과 알림 ON/OFF | P1 |

## 4. 비기능 요구사항

| ID | 항목 | 요구사항 |
|---|---|---|
| NF-01 | 성능 | 앱 실행 후 3초 이내 메인 화면 표시. 공고 목록 로딩 2초 이내 |
| NF-02 | 오프라인 | **2주 데모에서는 온라인 가정**. 인터넷 불안정 시 이전 로드된 공고 목록 캐시 표시는 2차 개발 |
| NF-03 | 접근성 | TalkBack/VoiceOver 기본 지원. 이미지 대체 텍스트 필수 |
| NF-04 | 보안 | 휴대폰 번호 외 민감정보 수집 금지. Firestore 보안 규칙 적용 |
| NF-05 | 다국어 | MVP 단계 한국어만 지원. 영어/중국어 확장성 고려한 구조 |

## 5. 화면 흐름 (User Flow)

```
[로그인/회원가입] → [프로필 등록]
    ↓
[메인 홈: 추천 공고 리스트]
    ↓
[필터 적용] → [공고 목록 필터링]
    ↓
[공고 상세 조회] → [지원하기 / 찜하기]
    ↓
[지원 완료 알림] → [마이페이지: 지원 내역]
```

## 6. 외부 연동

| 시스템 | 연동 방식 | 데이터 | 비고 |
|---|---|---|---|
| 고용24 Open API | Firebase Cloud Functions 프록시 | 채용 공고 목록, 상세 정보 | XML → JSON 변환, 2차 필터링 |
| Firebase Auth | SDK 직접 연동 | 휴대폰 인증, 사용자 식별 | |
| Firebase Firestore | SDK 직접 연동 | 사용자 프로필, 지원 내역, 찜 목록 | |
| FCM | SDK 직접 연동 | 푸시 알림 | |

## 7. 용어 정의

| 용어 | 정의 |
|---|---|
| 고용24 (GoYong24) | 고용노동부 산하 통합 일자리 플랫폼. 기존 워크넷 통합 |
| 세이프티 큐레이션 | 업무 강도를 시각적 배지로 사전 안내하는 기능 |
| 오터치 방어 | 노년층의 의도치 않은 다중 터치를 방지하는 UX 패턴 |
| P0 / P1 / P2 | 우선순위. P0=필수, P1=중요, P2=선택 |
