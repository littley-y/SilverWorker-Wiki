## 1. 전체 DoD (최종 데모 기준)

2주 후 아래 7개 흐름이 끊김 없이 동작해야 합니다:

| # | 기준 | 확인 방법 |
|---|---|---|
| 1 | 앱 실행 후 3초 이내 첫 화면 표시 | 스톱워치 측정 |
| 2 | 휴대폰 인증 완료 후 프로필 등록 가능 | 실제 번호로 SMS 수신 |
| 3 | 메인 화면에 공고 목록 최소 10개 표시 | 화면에서 카운트 |
| 4 | 필터 적용 시 목록 변경 | 지역/직종 칩 탭 후 결과 확인 |
| 5 | 공고 상세에서 업무 강도 배지 확인 | 세이프티 섹션 표시 확인 |
| 6 | 지원 버튼 클릭 후 Firestore에 저장 | Firebase Console에서 문서 확인 |
| 7 | 마이페이지에서 지원 내역 확인 | 지원 후 마이페이지 진입 |

## 2. 기능별 테스트 케이스

### AUTH (인증)

| ID | 시나리오 | 예상 결과 |
|---|---|---|
| T-AUTH-01 | 올바른 번호 → SMS 수신 → 코드 입력 → 프로필 등록 | 메인 홈 진입 |
| T-AUTH-02 | 앱 재실행 (로그인 상태) | 로그인 화면 없이 메인 진입 |
| T-AUTH-03 | 틀린 인증코드 입력 | "인증번호가 맞지 않습니다" 표시 |
| T-AUTH-04 | 로그아웃 → 앱 재실행 | 로그인 화면 표시 |

### JOB (공고)

| ID | 시나리오 | 예상 결과 |
|---|---|---|
| T-JOB-01 | 홈 진입 시 공고 목록 로드 | 카드 10개 이상 표시 |
| T-JOB-02 | 지역 필터 "종로구" 선택 | 종로구 공고만 표시 |
| T-JOB-03 | 직종 필터 "경비/관리" 선택 | 해당 직종 공고만 표시 |
| T-JOB-04 | 결과 없는 필터 조건 적용 | 빈 상태 화면 표시 |
| T-JOB-05 | 공고 카드 탭 | 상세 화면 진입 |
| T-JOB-06 | 상세 화면에서 세이프티 배지 확인 | physicalIntensity 등급 + 배지 표시 |

### APP (지원)

| ID | 시나리오 | 예상 결과 |
|---|---|---|
| T-APP-01 | 지원하기 → 자기소개 입력 → 제출 | 완료 화면 + Firestore 저장 |
| T-APP-02 | 지원 버튼 연속 3회 탭 | Firestore에 1건만 저장 |
| T-APP-03 | 이미 지원한 공고 재진입 | "이미 지원한 공고입니다" 표시 |
| T-APP-04 | 자기소개 200자 초과 입력 시도 | 200자에서 입력 차단 |

### MYPAGE (마이페이지)

| ID | 시나리오 | 예상 결과 |
|---|---|---|
| T-MY-01 | 마이페이지 진입 | 이름, 지역 표시 |
| T-MY-02 | 지원 내역 탭 | 지원한 공고 목록 표시 |
| T-MY-03 | 지원 내역 상태 배지 | "접수" 상태 배지 표시 |
| T-MY-04 | 로그아웃 | 로그인 화면으로 이동 |

### UI (시니어 UX)

| ID | 시나리오 | 예상 결과 |
|---|---|---|
| T-UI-01 | 전체 화면 폰트 크기 확인 | 14pt 미만 없음 |
| T-UI-02 | 주요 버튼 크기 확인 | 높이 56dp 이상 |
| T-UI-03 | 실제 기기 햅틱 | 버튼 탭 시 진동 감지 |
| T-UI-04 | 네트워크 차단 상태 | 에러 화면 + "다시 시도" 버튼 표시 |

## 4. 데모 시나리오 체크리스트 (Day 14 리허설용)

```
□ 앱 실행 → 스플래시 → 로그인 화면 (3초 이내)
□ 휴대폰 번호 입력 → SMS 수신 → 코드 입력
□ 프로필 등록 (이름: 김OO, 지역: 서울 종로구)
□ 메인 홈 공고 목록 표시
□ 필터: 지역 "종로구" → 직종 "경비/관리"
□ 공고 카드 탭 → 상세 화면
□ 세이프티 배지 확인 (서있기/보통)
□ "지원하기" 탭 → 자기소개 입력 → 제출
□ 지원 완료 화면 표시
□ Bottom Nav → 마이페이지 → 지원 내역 1건 확인
□ 로그아웃
```

예상 시연 시간: **3~5분**
